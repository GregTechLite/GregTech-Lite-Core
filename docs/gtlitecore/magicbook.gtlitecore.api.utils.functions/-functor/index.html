The functional interface Function is f: T -> R, which T is type of the param input, and R is type of return value. For example, if we want to transform a String to Integer, then we can define below: 


        Function<String, Integer> function = Integer::parseInt;

Then we should input a param "testString" in "domain" T, and then, the function Function<String, Integer> will map it to R, i.e. 

        Integer result = function.apply("testString");
Content copied to clipboard
In this code, Integer is R, and String is T. Functor is a higher version of Function.

Functor is a higher version of Function, T means the param type, such as: if T is Integer, then Functor<T, F> will consists of a integer value maybe; F is the type of Functor<T, F>, and another constraint is F must be the subtype of Functor<T, F>. The method map is not like apply, it will contain a function Function<T, R>, i.e. f: T -> R, and then map it to F, this method will transform value of Functor, return a new Functor which consists of the transformed value. For example, if we create a class which implements this class: 


        public class FunctorImpl implements Functor<String, Functor> {
            private String value;

            public FunctorImpl(String value) {
                this.value = value;
            }

            
            public <R> FunctorImpl map(Function<String, R> function) {
                R result = function.apply(value);
                return new Functor(result.toString());
            }
        }

        FunctorImpl functor = new FunctorImpl("testString");
